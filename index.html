Tempo Design System

Tempo is BDR Thermea's design system for efficiently creating and maintaining consistent user experiences throughout the product portfolio. BDR Thermea harbours many different brands, each with a different character. Tempo provides the single backbone for all digital experiences, for all BDR Thermea's brands. It ensures a consistent quality and reduces design and development efforts, while allowing brands to maintain their own unique character.

Brand values & principles

These are the BDR Thermea brand values, stemming from the company's vision.

  • Be reliable

    Show design excellence combined with technical excellence by being transparent about what is happening:

    • actionable advise Make the user understand what is happening and help them to get on with their task;
    • least surprise Some small positive surprises are good, but avoid negative surprises;
    • balance design Find common ground in design and architecture, and keep it consistent;
    • architecture stability Provide an environment where elements are behaving conveniently and intuitive, as opposed to distracting.

  • Be intuitive

    Make unambiguous choices by applying user-centered thinking and using a smart solutions approach, aiming for a multi-faceted ease of use:

    • to the edges Make the interface useful for the maximum number of users;
    • repetition & rythm Repeat (visual) elements in patterns for consistency and a feeling of harmony;
    • fit for purpose Think about goals and constraints of the user and make the interface suit the purpose;
    • less is more Keep elements as minimal as possible.

  • Be informative

    Communicate sustainability and comfort by providing the right information at the right time:

    • information scent Provide clear information in text and visual cues to guide the user on the right path;
    • context & feedback Communicate the big picture, and let users know what is happening;
    • transition design Provide the user with relevant information and actions to achieve (future) sustainability;
    • learnability The product learns from usage and can respond to this.

  • Be connected

    Provide seamless solutions and patterns in software and hardware:

    • integrated Bring together all touch points, including specialisms that are usually considered seperately;
    • elegance Even though functionally the interface is complex, the design is remarkably simple;
    • seamless There are no interruptions by technical limitations or design issues;
    • stay out of the way Let users do things the way they do, by avoiding unnecessary procedures.

Styleguide

The following section provides a reference for visual characteristics of BDR Thermea's digital products.

  • Colors

    In BDR Thermea's digital product colors are defined cross-brand and partly brand-specific. The brand-specific colors are applied as accent alongside dark, medium and light greys to differentiate the user interface between brands.

    Cross-brand: Colors applied to user interface elements that are shared among brands.

    Brand-specific: Colors applied to user interface elements to distinguish the brands.

    • Light & Dark themes

      When an appliance is equipped with a color screen, the graphical user interface can be based on both a light or a dark background. Light themes are applied to customer-facing controls, dark themes to controls that are aimed at professional users, such as service engineers.

    Components

    User interface elements applied in common scenarios. The application of components varies depending on the device; a touch screen demands a different approach than a regular screen controlled with physical buttons. And a touch screen on a handheld device like a phone or a tablet is different from a wall-mounted device with touch screen.

    • Button

    • Menu

      The menu is the main component used for navigating to specific places in the user interface. It therefore also serves as a map of a product's functionalities.

      • Definition Menu’s are logically ordered groups of items for a user to navigate;
      • Reuse Menu structures should be reused over devices as much as possible;
      • Order Menu items should be ordered based on priority and frequency of use. Prevent abbreviations and truncation. If unavoidable, apply consistent;
      • User-specific Distinguish installer items from end user items. If needed, installer items should be grouped seperately and limited for an end user to use;
      • Intuitive Menu’s are ordered logically and structured in a clear way to the user. Texts speak the language of the target group;
      • Consistency The name of a parent item equals the title/header for the child menu view;
      • Actionable Texts in menus are active and elicit action;
      • Relevance Menu items contain relevant and informative elements on a level as high as possible: a user should not need to drill down further than necessary to view data and settings. For instance a temperature value should be shown before changing a setting, not only after selection;
      • Layout Menu’s layout relate logically to input;
      • Feed forward Items that can be navigated further/deeper should communicate this and provide feedforward.

    Patterns

    Behavioral patterns applied in common scenarios.

    • Modal dialog

      Applied for performing specific actions in a seperate context. Returns to the original context after completing.

    • Wizard

      Applied for complex actions that will be performed infrequently.

    • Contextual help

      Provides clear and concise explanation of features or status when and where it applies; in the relevant context.
